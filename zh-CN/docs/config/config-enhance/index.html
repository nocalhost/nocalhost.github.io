在开发模式下自动转发端口#

name: nocalhost-api
serviceType: deployment
containers:
- name: nocalhost-api
dev:
portForward:
- 8080:80
- 3306:3306

如果你期望开发容器的某些端口能够在进入开发模式后自动端口转发到本地， 可以进行一些相关的配置。

关于系统端口

如果需要权限才能监听本地端口，例如 Ubuntu 或 Windows 的系统端口(低于1024) 进入开发模式后无法启用自动端口转发功能。


源码地址#

name: nocalhost-api
serviceType: deployment
containers:
- name: nocalhost-api
dev:
gitUrl: git@github.com:nocalhost/nocalhost.git

源代码地址是指服务的 git 源代码目录。 它用于快速和轻松地下载源代码，支持 http/https 和 ssh 协议。

tip

代码是否能够成功克隆取决于你在使用这个功能时使用的设备是否有权限。

适宜场景为提前配置好各个服务的 gitUrl， 这样，对于团队协作来说，可以省去团队之间服务与业务的沟通成本。


文件同步#

name: nocalhost-api
serviceType: deployment
containers:
- name: nocalhost-api
dev:
sync:
type: send
mode: pattern
filePattern:
- .
ignoreFilePattern:
- ".git"
- ".github"
- ".vscode"
- "node_modules"

文件同步配置包括三个部分：

  • 第一个是同步类型，包括：

note
  • send. 仅发送 (单向) 所有更改都取决于本地，远程更改不会影响本地(默认)。
  • sendReceive. 发送并接收(双向)。 任一端添加、修改和删除等操作将同步到另一端。

  • 第二是本地文件删除保护 containers[].dev.sync.deleteProtection, 这是一个布尔值。

      - 如果启用此功能， 在远程删除将不会同步到本地(默认启用此功能)。

      - 如果启用此功能时，在 sendreceive 类型的同步下，当文件在远程中被删除时，相应的本地文件也将被删除。

  • 第三个是同步模式。 当进入 DevMode 时，您需要选择一个本地目录，然后 Nocalhost 将默认同步此目录下的所有文件。 如果您不想同步所有文件，你可以定制同步模式。

Nocalhost 提供两种同步模式。 containers[].dev.sync.mode

note
  • pattern. 基于模式匹配的文件同步(默认模式)。
  • gitIgnore. 通过 gitIgnore来进行同步和忽略文件。

模式匹配#

如果你想使用模式匹配，你可以配置 filePattern 和 ignoreFilePattern， 例如只同步构建产物或忽略所有与构建无关的文件。

比如上面给出的例子，表示只发送同步文件，并且忽略 .git、.github、.vscode 和 node_modules 下的文件。

::::info 模式匹配

在 模式配置 中查看更多说明。

:::


gitIgnore 模式#

这种模式很容易使用。 它将自动使用 git 通过 gitignore 来控制文件忽略。

要求

因为这个功能是基于 git的，因此你需要确保您的设备已安装 git。 此外，同步目录必须是一个 git 项目。

如果不满足上述两个要求，Nocalhost 将不会启用同步控制，这意味着所有文件都将被同步。 这相当于没有进行同步配置。


哪些文件被忽略了？

您可以移动到文件同步目录(例如 cd /yourpath)， 然后输入 git status --ignored=matching -s 以查看忽略的文件/文件夹。 以 !! 开头的文件/文件夹将不会同步到遥控器。


示例：

name: nocalhost-api
serviceType: deployment
containers:
- name: nocalhost-api
dev:
sync:
type: send
mode: gitIgnore
